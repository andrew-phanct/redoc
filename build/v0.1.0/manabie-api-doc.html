Authentication

Salesforce API uses OAuth 2.0 JWT Bearer Token Flow for Server-to-Server integration. This authentication method is designed for server-to-server scenarios, bypassing user interaction. It allows a client application to directly authenticate to the Salesforce server using a digital certificate and a JSON Web Token (JWT) assertion.

The client application first signs a JWT with its private key, then sends the signed JWT in a request to the Salesforce server. The server verifies the JWT and issues an access token to the client application. The client application can then use the access token to authenticate to the Salesforce server on behalf of the user or the application itself.

For generating the JWT Bearer token, please refer to the Salesforce documentation for the full implementation.

Obtain an access token

Assume that we have already obtained a JWT bearer token from Salesforce Identity. This request is to obtain an access token using the JWT bearer token and the access token can then be used to authorize other API requests.

None
Request Body schema: application/x-www-form-urlencoded
required

required
string

Grant type parameter. Must be "urn:ietf:params:oauth:grant-type:jwt-bearer" for JWT bearer token flow.

assertion
required
string

The JWT bearer token.
